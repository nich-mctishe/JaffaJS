Controllers

Controllers are objects that handle all the requests that are being made. Be that requests for a new page, requests to the server, or simple population of a field, the controller will either either perform the request itself or use one of the many helper objects to complete the task for it.

Ideally Controllers should be split along each main operation. For example this Application has three controllers.

  • initial controller - each application will need a controller to begin performing functions on page load.
  • form controller - to handle to saving and displaying of the data on the form
  • manual controller - to assist with the displaying of the manual

to save on code many pages will not need a controller to function. a static page in JaffaJS will spawn using only the Loader.

each controller will need to the following code:

var myController = { onDeviceReady: function() { //code goes here to be called when the controller is first loaded }, controllerIcons: function(){ //this is where you should place all the calls to external 'pages' in this script so that their onDeviceReady functions will run. } }

All controllers are placed in the folder js/controllers/ and will need to be called from the index.html page inside <script></script> tags

please remember that all controller object need unique name, but the same controller can be called many times.
